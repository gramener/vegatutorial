Changing chart types -- area chart

This example shows how to plot areachart based on barchart example.

The Vega spec area.json is quite similar to how we constructed the barchart. Let's look at what's different.

The "scales" section defines how to map data values to visual values (pixel positions, colors etc.). Here, of two scales, X "xscale" needs few modifications. 

    "scales": [
      {
        "name": "xscale",              // Create a scale named "xscale"
        "type": "linear",              // on linear scale, and not ordinal
        "domain": {"data": "table",    //
                   "field": "x"},      //
        "range": "width",              //
        "zero": false                  // chart doesn't begin with zero scale
      },

The "axes" section for "x"axes we set the ticks

    "axes": [
      {"type": "x", "scale": "xscale", "ticks": 4},  // restrict ticks to 4 on x-axis

The "marks" type is defined as "area"

    "marks": [
      {
        "type": "area",                 // mark type is area

The "properties" have similar definitions as barchart. However, in areachart "width" under ("enter") needn't be defined like

    "enter": {
      "x": {"scale": "xscale", "field": "x"},
      "y": {"scale": "yscale", "field": "y"},
      "y2": {"scale": "yscale", "value": 0},
      "fill": {"value": "steelblue"}
